# telemetry_csv_validator

**Русский** | [English](README.md)

`telemetry_csv_validator` — Python-утилита для проверки CSV-файлов с промышленной телеметрией.

Утилита проверяет входящие записи телеметрии, разделяет корректные и ошибочные строки и записывает результат в два выходных файла:

- `clean.csv` — корректные записи телеметрии;
- `errors.csv` — ошибочные записи с дополнительной колонкой `error`, содержащей описание проблемы.

Проект предназначен для контроля качества данных перед загрузкой телеметрии в базы данных, ETL-процессы или аналитические хранилища.

---

## Формат входных данных

Входной CSV-файл должен содержать следующие колонки:

```csv
station_id,timestamp,pressure,temperature
```

Пример:

```csv
station_id,timestamp,pressure,temperature
PUMP_001,2026-07-03T10:15:00,6.4,35.2
PUMP_002,2026-07-03T10:16:00,6.5,35.3
PUMP_003,2026-07-03T10:17:00,7.0,36.1
```

---

## Правила проверки

Каждая строка проверяется по следующим правилам:

| Поле | Правило |
|---|---|
| `station_id` | Не должно быть пустым |
| `timestamp` | Должно быть корректным значением даты и времени в формате ISO 8601 |
| `pressure` | Должно быть числом от `0` до `100` |
| `temperature` | Должно быть числом от `-50` до `150` |

---

## Выходные файлы

### `clean.csv`

Содержит только корректные записи телеметрии.

Пример:

```csv
station_id,timestamp,pressure,temperature
PUMP_001,2026-07-03T10:15:00,6.4,35.2
PUMP_002,2026-07-03T10:16:00,6.5,35.3
```

### `errors.csv`

Содержит ошибочные записи и дополнительную колонку `error`.

Пример:

```csv
station_id,timestamp,pressure,temperature,error
,2026-07-03T10:16:00,6.5,35.3,station_id is empty
PUMP_002,bad_date,7.0,36.1,timestamp is invalid
PUMP_003,2026-07-03T10:18:00,150,34.0,pressure is out of range
```

## Сценарии использования

`telemetry_csv_validator` может использоваться для:

- проверки выгрузок телеметрии перед импортом в базу данных;
- разделения корректных и проблемных записей;
- подготовки данных для ETL-обработки;
- контроля качества данных в системах промышленной автоматизации;
- предварительной обработки CSV-файлов от PLC, SCADA, шлюзов или внешних источников телеметрии.

---

## Функции проверки

| Функция | Назначение |
|---|---|
| `validate_station_id` | Проверяет, что идентификатор станции не пустой |
| `validate_timestamp` | Проверяет, что timestamp является корректной ISO datetime-строкой |
| `validate_pressure` | Проверяет, что давление находится в допустимом диапазоне |
| `validate_temperature` | Проверяет, что температура находится в допустимом диапазоне |
| `validate_row` | Проверяет одну строку телеметрии целиком |
| `process_csv` | Обрабатывает входной CSV и записывает выходные файлы |

## Принципы разработки

## Тестирование

Для автоматических тестов используется `pytest`.

Планируемые области тестирования:

- корректные и некорректные значения `station_id`;
- корректные и некорректные ISO datetime-строки;
- граничные значения давления;
- граничные значения температуры;
- проверка полной строки;
- запись корректных строк в `clean.csv`;
- запись ошибочных строк в `errors.csv`.
